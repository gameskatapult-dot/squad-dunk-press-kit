# Editing the Squad Dunk press kit

Non-technical editors should edit this file:

`content/squad-dunk.yml`

Do not edit anything inside `build/`. The `build/` folder is generated automatically by GitHub Pages and local build commands.

## Quick text edits

Open `content/squad-dunk.yml` and edit:

- `description`: short product summary.
- `history`: longer product story.
- `features`: bullet list.
- `genres`: factsheet genre tags.
- `platforms`: factsheet platform entries.
- `team`: About Katapult Games, boilerplate, and credits text.
- `contacts`: press email and studio links.

Keep YAML indentation consistent. Use two spaces under lists and nested fields.

## Reorder sections

Move entries up or down in `sections`:

```yaml
sections:
  - id: factsheet
    enabled: true
  - id: description
    enabled: true
  - id: media
    enabled: true
```

The page preserves the current visual layout. `factsheet`, `description`, and `features` are rendered as one overview block. `team` and `contact` are rendered as one final two-card block.

## Hide a section or item

Set `enabled: false`.

Example:

```yaml
- id: reveal-trailer
  title: Reveal Trailer
  enabled: false
```

Disabled media items are not shown and their files are not required during validation.

## Reorder screenshots

Move screenshot blocks up or down under:

`media.screenshots.items`

The gallery follows the exact YAML order.

## Add a screenshot

1. Upload the image into:
   `squad-dunk/images/screenshots/`
2. Copy an existing screenshot block in `content/squad-dunk.yml`.
3. Update `id`, `file`, `thumb`, `title`, and `alt`.
4. Keep `enabled: true`.

Example:

```yaml
- id: new-action-shot
  file: images/screenshots/new-action-shot.png
  thumb: images/screenshots/new-action-shot.png.thumb.jpg
  title: New action shot
  caption: Optional short caption.
  alt: First-person basketball action on the Squad Dunk court.
  enabled: true
```

The `.thumb.jpg` file is generated during the press kit build. It is okay if it does not exist in the repo.

## Update download ZIP buttons

Download labels and paths are controlled in `downloads`:

```yaml
- id: screenshots-zip
  title: download all screenshots as .zip
  file: images/images.zip
  section: media
  generated: true
  enabled: true
```

`images/images.zip` and `images/logo.zip` are generated by presskit.html during the build. For any custom ZIP path, upload the ZIP file and set `generated: false` or remove `generated`.

## Branding assets

Branding order is controlled by:

`branding.assets`

Move entries up/down to reorder. Set `enabled: false` to hide an asset. If `enabled: true`, the referenced file must exist.

## Edit from GitHub web

1. Open `content/squad-dunk.yml` on GitHub.
2. Click the pencil icon.
3. Make the edit.
4. Click **Commit changes**.
5. GitHub Actions will regenerate `squad-dunk/data.xml`, rebuild the press kit, and deploy GitHub Pages automatically.

## Local commands

After editing locally, run:

```bash
npm run content
npm run build:presskit
```

`npm run content` validates the YAML and generates `squad-dunk/data.xml`.
